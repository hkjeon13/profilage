# 기업 프로필 상단 영역 개선 계획

## 1. 목적

기업 프로필 상단의 히어로 카드와 `기업 기본정보` 카드를 하나의 일관된 기업 개요 영역으로 유지하면서, 남은 시각적 불균형과 상호작용 문제를 정리한다.

초기 화면의 강한 파란색 장식, 평가 문구와 데이터 출처 배지는 1차 개선에서 제거되었고 좌우 영역도 하나의 카드로 통합되었다. 이번 후속 개선에서는 이니셜 박스와 기업명 묶음의 비례, 중복 시장 배지, 종업원 수 기준일 아이콘의 크기와 상호작용을 정리한다. 사용자는 기업명, 대표자, 설립일, 직원 수 등 핵심 사실을 빠르게 파악할 수 있어야 하며, 상장시장 정보는 상단 식별 영역에서 반복하지 않고 `상장 및 주가` 섹션에서 제공한다.

## 2. 대상 범위

- 좌측 `company-profile-card` 히어로 영역
- 우측 `profile-basic-card` 기업 기본정보 영역
- 두 영역을 감싸는 `profile-top-grid`
- 태블릿 및 모바일 반응형 레이아웃

다음 영역은 이번 작업에서 변경하지 않는다.

- 프로필 상단 내비게이션
- 검색 결과로 돌아가기 링크
- 프로필 섹션 내비게이션
- AI 기업요약 이하의 상세 콘텐츠
- 데이터 조회 및 API 처리 방식

`상장 및 주가` 섹션의 구조와 데이터 조회 방식은 변경하지 않는다. 다만 상단 시장 배지를 제거한 뒤에도 상장사·비상장사 상태가 해당 섹션에서 확인되는지는 회귀 검증한다.

## 2.1 구현 전 선행 조건

현재 운영 SSH 서버에는 로컬 작업 트리에 아직 통합되지 않은 프로필 UI 변경이 존재할 수 있다. 구현을 시작하기 전에 운영 서버, `origin/main`, 로컬 작업 트리의 다음 파일을 비교하고 최신 변경을 로컬 기준 소스에 안전하게 통합한다.

- `app/static/profile.html`
- `app/static/profile-page-5.js`
- `app/static/profile-chart-2.css`
- `app/static/styles.css`

기존 로컬 변경과 운영 서버 변경을 단순 덮어쓰지 않는다. 병합 후 JavaScript 문법 검사와 관련 테스트를 통과한 소스를 새로운 배포 기준으로 사용한다.

## 2.2 현재 상태와 남은 작업

이 문서는 전체 디자인 방향과 함께 **현재 운영 화면에서 아직 남은 변경만 구현하기 위한 기준**으로 사용한다. 이미 적용된 항목을 다시 별도 CSS 블록으로 추가하지 않는다.

| 항목 | 현재 상태 | 목표 상태 | 작업 구분 |
| --- | --- | --- | --- |
| 통합 상단 카드 | 좌우 영역이 하나의 외곽 카드 안에 배치됨 | 구조 유지 및 회귀 검증 | 검증 |
| 파란 히어로·평가 장식 | 제거됨 | 제거 상태 유지 | 검증 |
| 좌우 비율·기본정보 그리드 | 데스크톱 35:65, 기본정보 3열 적용됨 | 기존 구조 유지 | 검증 |
| 이니셜 박스 | 데스크톱 54px, 모바일 46px | 이름 묶음 높이에 맞춘 확정 크기로 확대 | 변경 |
| 상단 시장 배지 | 기업명 아래 `KOSPI` 등 표시 | 상단에서 DOM과 여백 모두 제거 | 변경 |
| 종업원 수 기준일 | 17×17px 정보 아이콘 툴팁으로 제공됨 | 시각 원 12px, 실제 hit area 24px 및 터치·키보드 동작 보강 | 변경 |
| 누락값 표시 | 기본정보에서 `-` 사용 | 데이터 상태표 기준으로 통일 | 변경 및 검증 |

CSS는 `styles.css` 마지막의 `/* Unified company identity and basic-information header. */` 블록을 기준 소스로 수정한다. 같은 선택자의 이전 정의 위에 새 오버라이드 블록을 추가하지 않으며, 안전하게 제거 가능한 중복 정의는 관련 테스트 확인 후 정리한다.

## 3. 핵심 설계 원칙

### 기업 정보 우선

장식이나 서비스 설명보다 기업명과 주요 사실이 먼저 보여야 한다.

### 하나의 프로필 헤더

좌우 카드가 서로 다른 컴포넌트처럼 보이지 않도록 배경, 테두리, 간격, 높이와 타이포그래피를 통일한다.

### 중복 제거

동일하거나 유사한 의미를 반복하는 문구와 필드를 제거한다.

- `공개 데이터`
- `Profilage`
- `Profilage 기업평가`
- `데이터`
- 상단 식별 영역의 상장시장 배지

`법인구분`은 현재 기본정보 렌더링 목록에 없으므로 이번 작업에서 다시 추가하지 않는다. 상장시장 데이터 역시 상단 카드에 중복 표시하지 않고 `상장 및 주가` 섹션의 기존 텍스트 배지에서만 제공한다.

### 정보 밀도 조절

셀마다 강한 테두리를 두기보다 여백과 얇은 구분선으로 정보를 나눈다.

## 4. 권장 정보 구조

### 전체 컨테이너

- `#ffffff` 배경의 통합 외곽 카드
- 데스크톱에서 `grid-template-columns: minmax(0, .54fr) minmax(0, 1fr)`을 사용해 좌측 약 35%, 우측 약 65%로 배치
- 외곽선 `1px solid #e4e7ec`, 모서리 반경 `12px`, 그림자 `0 12px 30px rgba(31, 42, 68, .07)` 사용
- 좌측 식별 영역 배경은 `#f8fafc`, 우측 기본정보 영역 배경은 `#ffffff` 사용
- 데스크톱에서는 좌우 영역 사이에 `1px solid #eaecf0` 세로 구분선을 사용하고 CSS Grid의 stretch로 높이를 맞춤
- 820px 이하 세로 배치에서는 같은 높이를 강제하지 않고 세로 구분선을 `1px solid #eaecf0` 가로 구분선으로 전환

구현 시 기존 JavaScript 연결점과 접근성 구조를 보존한다.

- `profile-top-grid`를 통합 카드의 외곽 컨테이너로 사용
- `company-profile-card`와 `profile-basic-card`의 클래스 및 ID 유지
- 두 자식 영역의 개별 배경, 외곽선과 그림자를 제거해 하나의 카드처럼 표현
- `#profile-market-badge`는 빈 요소로 숨겨두지 않고 마크업에서 제거하며 JavaScript 조회·할당 코드와 테스트도 함께 제거
- 데이터 바인딩에 사용되는 나머지 기존 선택자를 변경해야 한다면 JavaScript와 테스트를 함께 수정

### 좌측: 기업 식별 영역

표시 순서:

1. 기업명 기반 이니셜
2. 기업명
3. 영문명

세부 방침:

- 강한 파란색 전체 배경과 격자 장식 제거
- 기업명을 가장 큰 제목으로 사용
- 영문명은 보조 텍스트로 축소
- 좌측 식별 영역의 `KOSPI`, `KOSDAQ`, `비상장` 배지는 제거해 기업명 묶음의 시각적 밀도를 낮춤
- 상장시장 정보는 `#section-market`의 기존 시장 배지에서 계속 제공하며 데이터와 기능 자체는 제거하지 않음
- 이번 범위에서는 별도 외부 로고 소스를 추가하지 않고 기업명 기반 이니셜만 사용
- 이니셜은 현재 `initials()` 정책을 유지해 괄호 표현을 제거한 기업명의 첫 유효 문자 최대 2개만 표시
- 이니셜 박스는 JavaScript로 텍스트 높이를 측정하지 않고 브레이크포인트별 고정 정사각형으로 구현
- 821px 이상은 72×72px, 561~820px은 64×64px, 560px 이하는 56×56px 사용
- 위 크기는 한글명과 영문명이 각각 한 줄인 정상 상태의 전체 텍스트 높이에 맞춘 값이며 최소·최대 크기도 동일한 값으로 고정
- 긴 기업명이나 영문명이 줄바꿈되더라도 이니셜 박스는 커지지 않고 텍스트만 아래로 확장
- 이니셜과 기업명 묶음은 상단 기준으로 정렬하고, 이니셜 글자는 박스 안에서 수직·수평 중앙 정렬
- 이니셜 박스와 기업명 묶음 사이 간격은 모든 브레이크포인트에서 16px 사용
- 이니셜은 현재보다 대비를 낮추고 장식 요소로 처리
- 서비스 홍보성 문구 및 평가 박스 제거

### 우측: 기업 기본정보 영역

주요 정보:

- CRNO
- 대표자
- 설립일
- 상장일
- 종업원 수 및 기준일
- 본사주소
- 홈페이지

세부 방침:

- `기업 기본정보` 제목은 의미와 문서 구조를 위해 `<h2>`로 유지하되 시각적 크기를 낮춤
- 핵심 정보는 확정된 반응형 기준에 따라 1열, 2열 또는 3열 그리드로 배치
- `법인구분`은 현재와 같이 표시 목록에 포함하지 않음
- 본사주소와 홈페이지는 하단 전체 너비 행으로 구성
- 값이 없는 경우 `-`로 통일
- 홈페이지는 외부 링크임을 아이콘과 접근성 라벨로 안내
- 종업원 수 기준일은 값 옆에 직접 표시하지 않고 `종업원수` 라벨 옆 정보 아이콘의 툴팁으로 제공
- 종업원 수 정보 아이콘의 시각 크기는 12×12px, `i` 글자는 7~8px로 사용해 필드 라벨보다 강조되지 않도록 함
- 정보 아이콘과 라벨 사이 간격은 3px로 제한
- 작은 시각 크기와 별개로 hover·focus를 위한 실제 상호작용 영역은 최소 24×24px 확보

### 데이터 표시 상태표

| 필드 또는 상태 | 표시 규칙 |
| --- | --- |
| 일반 문자열 | `null`, `undefined`, 빈 문자열, 공백 문자열이면 `-` |
| 기업명 | 누락 시 `기업명 정보 없음`을 제목으로 표시 |
| 영문명 | 누락 시 `#profile-subtitle`을 숨기고 해당 여백을 제거 |
| CRNO | 기업 개요, DART, URL 파라미터 순서로 대체값을 사용하고 모두 없으면 `-` |
| 대표자·설립일·상장일·본사주소 | 유효한 값이 없으면 `-` |
| 종업원 수 | 숫자로 변환되고 0보다 클 때만 천 단위 구분해 표시하며 0, 음수, `NaN`, 누락은 `-` |
| 종업원 수 기준일 | 유효한 종업원 수와 기준일이 모두 있을 때만 정보 아이콘을 표시하고, 둘 중 하나라도 없으면 아이콘과 툴팁을 렌더링하지 않음 |
| 홈페이지 | `http://` 또는 `https://`로 정규화 가능한 주소만 새 창 링크로 표시하고 유효하지 않거나 누락되면 `-` |
| 상장시장 | 상단 카드에서는 렌더링하지 않으며 `#section-market`에서 상장사는 시장명, 비상장사는 `비상장` 또는 `주가 정보 미제공` 상태를 텍스트로 표시 |
| 로딩 상태 | 기존 제목·영문명·기본정보 스켈레톤을 유지하고 시장 배지 자리나 빈 여백을 만들지 않음 |
| 오류 상태 | 기존 오류 제목과 기본정보 오류 메시지를 유지하며 이니셜은 `P`, 시장 배지는 표시하지 않음 |

## 5. 시각 스타일

### 색상

- 통합 카드 배경: `#ffffff`
- 좌측 식별 영역 배경: `#f8fafc`
- 기본 텍스트: `#101828`
- 보조 텍스트: `#667085`
- 구분선: `#eaecf0`
- 이니셜 박스 테두리: `#d0d5dd`
- 링크와 포커스 강조색: `#1570ef`
- 장식 목적의 그라데이션과 넓은 파란 면적은 제거

### 타이포그래피

- 기업명: 상단 영역의 유일한 대형 제목. 821px 이상 `36px/42px`, 561~820px `30px/36px`, 560px 이하 `24px/28px`
- 영문명: 821px 이상 `14px/22px`, 820px 이하 `13px/20px`; 기업명과의 간격은 8px
- 이니셜 글자: 821px 이상 18px, 561~820px 17px, 560px 이하 15px; 굵기 700
- 기본정보 제목: `13px`, 굵기 800, 글자 간격 `.02em`
- 필드 라벨: `11px`, 굵기 750, 색상 `#667085`
- 필드 값: `14px`, 굵기 650, 색상 `#101828`
- 긴 주소와 영문명은 자연스럽게 줄바꿈

### 테두리와 간격

- 통합 카드 외곽선, 반경과 그림자는 `전체 컨테이너`의 확정값 사용
- 좌측 내부 여백: 821px 이상 32px, 561~820px 26px, 560px 이하 20px
- 우측 내부 여백: 821px 이상 28px 30px, 561~820px 24px 26px, 560px 이하 18px 20px
- 기본정보 열 간격: 821px 이상 24px, 561~820px 22px, 560px 이하 16px
- 각 기본정보 행은 라벨과 값 사이 6px, 상하 15px 패딩, 하단 `1px solid #eaecf0` 구분선 사용
- 개별 정보 셀에는 외곽 박스, 배경색 또는 세로 테두리를 추가하지 않음

### 정보 아이콘과 툴팁

- 정보 아이콘은 유효한 종업원 수와 기준일이 모두 있는 경우에만 사용
- 인터랙션 요소는 `<button type="button" class="profile-field-info">`로 렌더링하고 브라우저 기본 버튼 스타일을 초기화
- 버튼의 실제 크기는 24×24px로 유지하되, 중앙의 시각적 원만 12×12px, `1px solid #98a2b3`로 표현
- 아이콘 내부 `i`는 8px, 굵기 700, 색상 `#667085`로 표시
- 버튼의 실제 상자와 라벨 사이에는 음수 보정 또는 내부 정렬을 사용해 시각적 원 기준 간격이 3px로 보이도록 함
- 각 툴팁에 고유 ID를 부여하고 버튼에 `aria-describedby`, `aria-expanded`를 연결
- 기본 상태에서는 툴팁을 숨기고 레이아웃 공간을 차지하지 않도록 함
- 마우스 hover와 키보드 focus에서 열고, 클릭 또는 터치 탭으로 열림 상태를 토글
- `Escape`, 다른 곳 클릭, 다른 정보 아이콘 열기 시 기존 툴팁을 닫고 `aria-expanded="false"`로 복원
- 툴팁 너비는 내용 기반으로 하되 `max-width: min(190px, calc(100vw - 40px))`, `white-space: normal`을 사용
- 툴팁은 카드 좌우 경계와 최소 12px 간격을 유지하도록 위치를 보정하며 360px 화면에서도 잘리지 않아야 함
- 동적으로 렌더링되는 기본정보 카드에 대응하도록 `profileBasicCard` 또는 문서 수준 이벤트 위임을 사용

## 6. 반응형 동작

### 데스크톱

- 821px 이상에서 `.54fr 1fr` 좌우 배치
- 이니셜 박스 72×72px
- 기본정보는 `repeat(3, minmax(0, 1fr))` 3열 정보 그리드
- 주소와 홈페이지는 전체 너비
- 좌우 사이에는 세로 구분선 사용

### 태블릿

- 561~820px에서는 기업 식별 영역과 기본정보 영역을 세로로 전환
- 이니셜 박스 64×64px
- 기본정보는 `repeat(2, minmax(0, 1fr))` 2열 그리드
- 기업명과 영문명이 기본정보보다 먼저 읽히도록 순서 유지
- 좌우 세로 구분선을 제거하고 기업 식별 영역 하단에 가로 구분선 사용

### 모바일

- 560px 이하에서는 기업 식별 영역과 기본정보 영역을 세로로 배치
- 이니셜 박스 56×56px
- 통합 카드 내부에서 별도 카드처럼 분리하지 않음
- 361~560px에서는 기본정보를 `repeat(2, minmax(0, 1fr))`로 유지하고 주소와 홈페이지는 1열 전체 너비
- 360px 이하에서만 기본정보를 1열로 전환
- 콘텐츠 길이에 따른 암묵적 열 전환이나 `auto-fit`은 사용하지 않음
- 긴 기업명, 대표자명, 주소와 홈페이지에는 `overflow-wrap: anywhere`를 적용해 정해진 열 수 안에서 줄바꿈

### 반응형 기준 요약

- 821px 이상: 좌우 `.54fr 1fr`, 로고 72px, 기본정보 3열, 세로 구분선
- 561~820px: 세로 배치, 로고 64px, 기본정보 2열, 가로 구분선
- 361~560px: 세로 배치, 로고 56px, 기본정보 2열, 가로 구분선
- 360px 이하: 세로 배치, 로고 56px, 기본정보 1열, 가로 구분선

## 7. 접근성

- 기업명을 페이지의 핵심 제목 구조와 일치시킴
- 이니셜은 기업명과 정보가 중복되는 장식 요소이므로 `aria-hidden="true"` 유지
- 좌측 시장 배지를 제거하더라도 `#section-market`에서 상장시장 또는 비상장 상태를 텍스트로 확인할 수 있어야 함
- 홈페이지 링크에 명확한 포커스 스타일 적용
- 종업원 수 정보 버튼에 의미가 분명한 `aria-label`, `aria-describedby`, `aria-expanded`와 키보드 focus 지원
- 시각적으로 작은 정보 아이콘도 최소 24×24px의 상호작용 영역을 제공
- 정보 버튼은 `Enter`와 `Space`로 열고 `Escape`로 닫을 수 있어야 하며 닫은 뒤 포커스를 유지
- 툴팁의 `role="tooltip"`과 연결 ID는 페이지 내에서 중복되지 않아야 함
- 텍스트와 배경의 명암비를 WCAG AA 수준으로 유지
- 화면 확대 200%에서도 정보 순서와 줄바꿈이 깨지지 않아야 함
