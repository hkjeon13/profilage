# 상위 기업집단 주주 역조회 설계

## 목표

기업 프로필 페이지에서 최대주주 또는 주요 주주를 클릭했을 때, 해당 주주의 현재 회사 기준 보유 정보와 상위 20개 기업집단 DB 안에서 발견되는 다른 보유 내역을 함께 보여준다.

기업집단 DB는 수동 목록이 아니라 공정거래위원회 공식 기업집단 데이터를 기준으로 동기화한다. 첫 운영 범위는 최신 지정월 기준 자산총액 상위 20개 공시대상기업집단이다.

## 공식 데이터 범위

상위 기업집단 기준 데이터는 공정거래위원회와 기업집단포털의 공식 데이터를 사용한다.

- 원천 계열: 공정거래위원회 기업집단포털 OpenAPI
- 기업집단 seed: `presentnYear` 기준 지정 대규모기업집단 목록
- 소속회사 seed: 선택된 기업집단별 소속회사 목록
- 상위 20개 산정 규칙: 최신 지정월 기준으로 공식 순위 또는 자산총액 기준 정렬
- 기업집단 목록 API에 순위나 자산총액 필드가 없으면 같은 지정월의 공정위 지정 결과 첨부자료로 보강

DART 데이터는 회사 중심 구조를 유지한다.

- 회사명, 종목코드, 법인등록번호 중 사용 가능한 값으로 DART `corp_code`를 매핑한다.
- 정기보고서 최대주주 현황은 `hyslrSttus`에서 수집한다.
- 임원 현황은 `exctvSttus`에서 수집한다.
- 임원 및 주요주주 소유보고는 신뢰도 향상이 필요한 후속 단계에서 추가한다.

이 기능은 “특정 인물이 한국 전체에서 보유한 모든 주식”을 보여준다고 주장하지 않는다. 범위는 “동기화된 상위 20개 기업집단 DB 안에서 발견된 동일명 또는 동일 법인 후보”로 제한한다.

## 사용자 경험

프로필 페이지의 기존 최대주주 영역은 유지한다. 표시되는 주주 행은 클릭 가능하게 만든다.

주주를 클릭하면 데스크톱에서는 우측 패널, 모바일에서는 모달 시트를 연다. 패널에는 다음 정보를 표시한다.

- 주주 표시명
- 엔티티 유형: 개인, 법인, 미상
- 현재 회사 기준 보유 비율, 관계, 보고서 기준
- 접수번호나 보고서 출처가 있으면 DART 출처 링크
- 상위 20개 기업집단 DB에서 발견된 다른 보유 후보
- 매칭 신뢰도
- 이름만으로 매칭된 경우 동명이인 가능성 안내

법인 주주는 인물처럼 표현하지 않는다. 주주가 법인으로 판단되면 “법인 주주” 문구를 사용하고, 법인등록번호, 종목코드, DART corp code 중 하나로 회사 매핑이 가능할 때 해당 회사 프로필로 연결한다.

## 백엔드 구성

### 공정위 기업집단 동기화

최신 공식 기업집단 범위를 동기화하는 서비스를 추가한다.

1. 사용 가능한 최신 지정월을 확인한다.
2. 해당 지정월의 공시대상기업집단 목록을 가져온다.
3. 공식 순위 또는 자산총액 기준으로 정렬한다.
4. 상위 20개 기업집단만 선택한다.
5. 각 기업집단의 소속회사 목록을 가져온다.
6. 원본 payload와 정규화된 row를 저장한다.

동기화는 멱등적이어야 한다. 같은 지정월을 다시 실행해도 중복 row를 만들지 않고 변경된 값만 갱신한다.

### DART 주주 인덱싱

저장된 기업집단 소속회사 목록을 기반으로 DART 데이터를 수집한다.

1. 각 회사를 DART corp code로 매핑한다.
2. 유효한 최신 보고기간의 최대주주 현황을 가져온다.
3. 같은 회사와 보고기간의 임원 현황을 가져온다.
4. 원본 DART payload는 기존 grouped data 저장 패턴을 활용해 보관한다.
5. 역조회에 필요한 주주 엔티티와 보유 내역 row를 정규화해 저장한다.

첫 구현은 수동 CLI 또는 관리자 명령으로 실행할 수 있게 한다. 데이터 형태가 검증된 뒤 스케줄러를 붙인다.

### 역조회 API

프로필 UI용 API를 추가한다.

- `GET /company/shareholders/{holder_id}`: 현재 주주 상세와 출처 기반 식별 필드를 반환한다.
- `GET /company/shareholders/search?name=...&corp_code=...`: 현재 회사 문맥을 선택적으로 받아 상위 20개 기업집단 DB 안의 주주 후보를 검색한다.

API 응답에는 매칭 신뢰도를 반드시 포함한다.

- `high`: 법인등록번호, DART corp code, 종목코드처럼 지속적인 식별자로 매칭
- `medium`: 정규화된 법인명과 회사 문맥으로 매칭
- `low`: 정규화된 개인명만으로 매칭

개인명만으로 매칭된 결과에는 동명이인 가능성 안내를 포함한다.

## 매칭 규칙

한국어와 영어 이름은 앞뒤 공백을 제거하고, 법인 비교 시 흔한 법인 접미사를 정규화한다. 원래 표시명은 그대로 보존한다.

엔티티 분류 규칙은 다음과 같다.

- 법인: 법인등록번호, DART corp code, 종목코드가 있거나 이름에 법인 접미사가 있는 경우
- 개인: 임원 현황과 매칭되거나 법인 근거가 없는 주주 row
- 미상: 개인/법인을 판단할 근거가 부족한 경우

개인 매칭은 보수적으로 처리한다. 이름만 같은 결과는 탐색 후보로 보여줄 수 있지만 신뢰도는 `low`로 표시한다. UI는 low 신뢰도 후보를 하나의 확정 인물로 병합하지 않는다.

## 오류 처리

공정위 동기화가 실패하면 마지막 성공 corpus를 유지한다. 동기화 실행 상태에는 시작 시각, 종료 시각, 기준 지정월, row 수, 오류 메시지를 저장한다.

DART 동기화 실패는 회사 단위로 처리한다. 한 회사 수집 실패가 상위 20개 전체 동기화를 중단시키면 안 된다.

프로필 UI는 다음 상태에서 자연스럽게 degrade한다.

- corpus가 아직 없으면 현재 회사 기준 주주 상세만 표시한다.
- 역조회 DB가 오래됐으면 갱신 시각과 stale 상태를 표시한다.
- 동일명 후보가 있으면 확정 보유 내역이 아니라 후보로 표시한다.

## 테스트

백엔드 테스트:

- 공정위 기업집단 응답 정규화
- 소속회사 응답 정규화
- 상위 20개 기업집단 선택
- DART 주주 데이터를 엔티티와 보유 내역으로 정규화
- 법인, 개인, 미상 엔티티 분류
- 역조회 신뢰도 산정
- 동기화 멱등성

프론트엔드 테스트:

- 주주 행 클릭 시 상세 패널이 열린다.
- 법인 주주 문구와 개인 주주 문구가 다르게 표시된다.
- low 신뢰도 동일명 안내가 표시된다.
- 역조회 결과가 비어 있어도 기존 최대주주 차트가 깨지지 않는다.

수동 검증:

- 최신 공식 지정월 기준으로 동기화를 실행한다.
- 삼성전자 프로필을 연다.
- 최대주주 행을 클릭한다.
- 현재 회사 기준 주주 상세가 표시되는지 확인한다.
- DB에 데이터가 있을 때만 다른 기업집단 내 후보가 표시되는지 확인한다.

## 구현 단계

Phase 1: 스키마와 공정위 동기화

- 공공데이터포털 service key 설정을 추가한다.
- 공정위/기업집단포털 클라이언트와 동기화 서비스를 추가한다.
- 정규화 테이블과 migration을 추가한다.
- 수동 동기화 명령을 추가한다.
- 순위 산정 fallback은 공식 순위 필드, 공식 자산총액 필드, 같은 지정월의 공정위 지정 결과 첨부자료 순으로 적용한다.
- 세 방법 모두 사용할 수 없으면 임의로 상위 20개를 고르지 않고 동기화 실행을 실패 처리한다.

Phase 2: DART 주주 인덱싱

- 기업집단 소속회사를 DART corp code로 매핑한다.
- 최대주주 현황과 임원 현황을 가져와 정규화한다.
- 주주 엔티티와 보유 내역을 저장한다.

Phase 3: 프로필 API와 UI

- 주주 상세/검색 API를 추가한다.
- 최대주주 row를 클릭 가능하게 만든다.
- 출처와 신뢰도 상태를 포함한 상세 패널을 추가한다.

Phase 4: 운영

- 동기화 실행 상태를 저장한다.
- 스케줄러 또는 문서화된 cron 명령을 추가한다.
- 원격 서버 배포와 공개 페이지 검증을 진행한다.

## 확정 범위

구현은 동기화 시점에 공식 소스에서 확인되는 최신 지정월을 사용한다. 첫 UI 릴리스는 상위 20개 기업집단 corpus 안의 후보만 보여준다. 상위 20개 밖의 전체 기업 검색은 이번 기능 범위에서 제외한다.
