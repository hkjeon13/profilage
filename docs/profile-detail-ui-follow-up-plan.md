# 기업 프로필 상세 UI 후속 개선 계획

## 1. 문서 목적

기업 프로필 상세 화면에서 브라우저 피드백으로 확인된 소규모 UI 개선을 누적 관리한다. 각 항목은 요청 내용, 구현 기준, 반응형 동작, 접근성, 변경 파일과 완료 조건을 함께 기록해 후속 구현과 배포 시 해석 차이가 없도록 한다.

## 2. 적용 범위

- 기업 프로필 상세 화면 `/profile`
- AI 기업요약 아래의 상세 섹션
- 데스크톱, 태블릿 및 모바일 반응형 상태
- 관련 JavaScript 렌더링 템플릿, CSS 및 회귀 테스트

상단 기업 개요 영역은 `docs/profile-top-section-redesign-plan.md`, 기업 심화 정보 영역의 대규모 재설계는 `docs/company-insight-section-redesign-plan.md`에서 별도로 관리한다.

## 3. 공시 이벤트 메타 정보 순서 변경

### 요청

공시 이벤트 행의 메타 정보가 현재 `배지 → 날짜` 순서로 표시된다. 사용자가 시간 정보를 먼저 읽을 수 있도록 `날짜 → 배지` 순서로 변경한다.

### 목표 구조

각 공시 이벤트 행은 다음 순서로 표시한다.

1. 날짜
2. 공시 유형 배지
3. 공시 제목
4. 요약 버튼

예시:

```text
2026.07.10  지분/최대주주  주식등의대량보유상황보고서(일반)  요약
```

### 구현 기준

- `renderDisclosureEventTimeline()` 템플릿에서 `.disclosure-event-date`를 `.disclosure-event-badge`보다 먼저 렌더링
- 데스크톱 그리드는 `76px 84px minmax(0, 1fr)` 사용
- 날짜는 76px 열에서 줄바꿈 없이 표시
- 배지는 84px 열에서 기존 최소 너비와 중앙 정렬 유지
- 제목은 남은 너비를 사용하고 기존 말줄임 처리 유지
- 요약 버튼은 제목 행 내부의 우측 동작 영역 유지
- 데이터, 정렬 기준, 공시 유형 색상과 클릭 기능은 변경하지 않음

### 반응형 기준

- 데스크톱: `날짜 76px → 배지 84px → 제목/요약` 한 줄 배치
- 모바일: 첫 행을 `날짜 76px → 배지 minmax(84px, 1fr)`로 배치
- 모바일 제목과 요약 버튼은 기존과 같이 다음 행 전체 너비 사용
- 360px 화면에서 날짜, 배지, 제목 또는 요약 버튼이 카드 밖으로 잘리지 않아야 함

### 접근성

- DOM 읽기 순서도 시각적 순서와 동일하게 `날짜 → 배지 → 제목 → 요약`을 유지
- 공시 제목 버튼과 요약 버튼의 기존 접근 가능한 이름을 유지
- 색상만으로 공시 유형을 구분하지 않고 배지 텍스트를 계속 제공

### 완료 조건

- 데스크톱에서 날짜 열 76px, 배지 열 84px이 적용됨
- 모바일에서 `76px minmax(84px, 1fr)` 메타 행이 적용됨
- 공시 제목 열림, 요약 모달, 말줄임과 이벤트 색상에 회귀가 없음

## 4. DART 상세정보 버튼 테두리 제거

### 요청

`DART 핵심정보` 제목 우측의 더보기 아이콘 버튼이 작은 테두리 카드처럼 보여 제목 영역에서 불필요하게 강조된다. 버튼의 기본 테두리를 제거해 가벼운 아이콘 동작으로 표현한다.

### 구현 기준

- 대상 선택자: `.dart-insight-more-button`
- 기본 상태에서 `border: 0`, `background: transparent` 적용
- 기존 38×38px 버튼 크기와 20×20px 아이콘 크기는 유지
- 아이콘 색상은 기존 `#185abc`를 유지
- hover 상태에서만 `#f2f4f7` 중립 배경을 표시해 클릭 가능성을 안내
- active 상태는 hover보다 한 단계 진한 중립 배경을 사용
- 버튼 기능, `aria-label`, `title`, 모달 연결은 변경하지 않음

### 접근성

- 테두리를 제거하더라도 `:focus-visible`의 `2px solid #1570ef` 포커스 링은 유지
- 키보드와 터치 사용자를 위해 실제 동작 영역은 최소 38×38px 유지
- 색상 변화에만 의존하지 않도록 접근 가능한 이름 `DART 상세정보 더보기` 유지

### 변경 파일

| 파일 | 변경 내용 |
| --- | --- |
| `app/static/styles.css` | 기본 테두리 제거 및 hover·active·focus-visible 상태 정의 |
| `tests/test_company_affiliate_api.py` | 기본 `border: 0`, 크기와 포커스 스타일 회귀 검증 |

### 완료 조건

- 기본 상태에서 버튼 외곽 테두리가 보이지 않음
- hover·active 상태에서만 중립 배경 피드백이 나타남
- 키보드 포커스 링이 명확하게 표시됨
- 더보기 모달 열기와 포커스 복귀 기능에 회귀가 없음

## 5. 기업 심화 정보 컬럼 구분선

### 요청

`기업 심화 정보`는 하나의 카드 안에서 다음 두 정보 그룹을 데스크톱 2열로 보여준다.

- 좌측 컬럼: `최대주주`, `감사의견`
- 우측 컬럼: `배당`, `재무비율`

두 컬럼의 소속 관계를 더 빠르게 인식할 수 있도록 카드 내부 중앙에 세로 구분선을 추가한다. 외곽 카드는 하나로 유지하며 각 항목을 별도 카드로 분리하지 않는다.

### 목표 구조

```text
┌─────────────────────────────────────────────────────────┐
│ DART 핵심정보                                      ··· │
│                                                         │
│ 최대주주                 │ 배당                         │
│                          │                              │
│ 감사의견                 │ 재무비율                     │
└─────────────────────────────────────────────────────────┘
```

### 구현 기준

- `renderCompanyInsightCards()`에서 단순 4개 카드 배열 대신 의미별 컬럼 래퍼를 생성
- 좌측 `.company-insight-column`: 최대주주 → 감사의견
- 우측 `.company-insight-column`: 배당 → 재무비율
- `.company-insight-card-grid`는 821px 이상에서 `repeat(2, minmax(0, 1fr))` 유지
- 컬럼 사이 간격은 0으로 두고 좌측 컬럼 `padding-right: 28px`, 우측 컬럼 `padding-left: 28px` 적용
- 우측 컬럼에 `border-left: 1px solid #eaecf0`을 적용해 카드 내부 높이에 맞는 하나의 연속 구분선 표현
- 각 컬럼 내부 항목 사이 간격은 24px 사용
- 기존 `.company-insight-card`의 투명 배경과 무테두리 스타일 유지
- 출처 문구는 두 컬럼 아래 전체 너비에서 기존대로 표시

### 누락 데이터 처리

- 특정 세부 항목이 없으면 해당 항목만 렌더링하지 않고 같은 컬럼의 나머지 항목은 유지
- 좌우 컬럼 중 한쪽에 표시할 항목이 전혀 없으면 단일 컬럼으로 확장하고 중앙 구분선을 표시하지 않음
- 구분선 위치를 `nth-child` 순서에 의존시키지 않고 명시적인 좌·우 컬럼 래퍼에 적용

### 반응형 기준

- 821px 이상: 2열과 중앙 세로 구분선 표시
- 820px 이하: 컬럼을 1열로 쌓고 세로 구분선, 좌우 패딩을 제거
- 모바일 적층 순서는 `최대주주 → 감사의견 → 배당 → 재무비율`
- 모바일 그룹 사이에는 기존 여백을 사용하며 새로운 외곽선이나 개별 카드 테두리를 추가하지 않음

### 접근성

- 구분선은 장식 요소이므로 접근성 트리에 노출하지 않음
- DOM 읽기 순서는 모바일 표시 순서와 동일하게 유지
- 각 세부 영역의 기존 제목 구조와 최대주주 버튼의 접근 가능한 이름 유지

### 변경 파일

| 파일 | 변경 내용 |

| `app/static/styles.css` | 데스크톱 세로 구분선, 컬럼 패딩과 모바일 해제 규칙 |

### 완료 조건

- 데스크톱에서 최대주주·감사의견과 배당·재무비율 사이에 1px 연속 구분선이 표시됨
- 구분선 양쪽에 각각 28px 내부 여백이 적용됨
- 외곽은 하나의 `기업 심화 정보` 카드로 유지됨
- 820px 이하에서 구분선과 불필요한 좌우 패딩이 제거됨
- 한쪽 컬럼 데이터가 모두 없을 때 구분선 없이 전체 너비를 사용함
- 최대주주 상세 버튼, DART 더보기 모달과 데이터 표시 기능에 회귀가 없음
