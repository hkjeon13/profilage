# 기업 심화 정보 영역 리디자인 계획

## 1. 목적

현재 `DART 핵심정보` 영역을 기존 2×2 카드 모음에서 벗어나, 사용자가 기업의 지배구조·주주환원·감사 신뢰도·재무건전성을 빠르게 판단할 수 있는 **기업 심화 신호 보드**로 재구성한다.

현재 화면은 최대주주 그래프, 배당, 감사의견, 재무비율이 동일한 시각적 무게로 나열되어 정보의 중요도와 관계가 드러나지 않는다. 작은 카드와 테두리가 반복되고, 단위가 없는 수치와 넓은 빈 공간이 혼재해 스캔 효율도 낮다.

## 2. 대상 범위

- `company-insight-summary-card`
- `renderCompanyInsightCards()`
- 최대주주 요약 및 주주 상세 진입
- 배당, 감사의견, 재무비율 표시
- 우측 상단 DART 상세정보 더보기
- 태블릿 및 모바일 반응형 표현

다음 항목은 이번 작업에서 변경하지 않는다.

- DART API 요청 방식과 응답 스키마
- 주주 상세 모달의 검색 기능
- DART 통합 상세 모달의 주식 구조·임직원 데이터
- 주의 신호 및 관계회사 영역
- 재무 요약과 공시 이벤트 영역

## 3. 핵심 사용자 질문

새 화면은 다음 질문에 빠르게 답해야 한다.

1. 지분이 특정 주주에게 얼마나 집중되어 있는가?
2. 주요 주주는 누구이며 각각 몇 퍼센트를 보유하는가?
3. 배당 정책을 판단할 수 있는 데이터가 있는가?
4. 감사의견과 감사인은 누구인가?
5. 부채비율, 수익성 등 핵심 재무비율은 어떠한가?
6. 더 자세한 주식 구조와 임직원 정보는 어디에서 확인하는가?

## 4. 새로운 정보 구조

### 4.1 헤더

- 제목을 `기업 심화 정보`로 변경
- 보조 문구로 `DART 정기보고서 기준`과 기준 사업연도 표시
- 우측 상단에 기존 DART 상세정보 더보기 아이콘 유지
- 더보기 아이콘은 주식 구조와 임직원 정보를 함께 제공하는 통합 모달을 연다

### 4.2 핵심 신호 요약 띠

헤더 아래에 가장 중요한 네 가지 신호를 한 줄 요약으로 제공한다.

- 최대주주: 최대주주명과 지분율
- 주요주주 집중도: 표시된 주요주주 지분율 합계
- 감사의견: 적정, 한정, 부적정, 의견거절 등
- 부채비율: 값과 단위 `%`

표현 원칙:

- 네 개 값을 동일한 작은 카드로 반복하지 않고 하나의 가로 요약 띠로 구성
- 라벨은 작게, 값은 명확하게 표시
- 의미가 정의되지 않은 임의의 정상·위험 색상은 사용하지 않음
- 데이터가 없으면 `-`로 표시

### 4.3 본문 왼쪽: 주요주주 구조

기존 다색 누적 막대와 긴 범례를 다음 구조로 교체한다.

- 주요주주를 지분율 내림차순으로 정렬한 순위 목록
- 각 행에 순위, 주주명, 지분율, 짧은 수평 막대 표시
- 주주명은 기존 상세 모달을 여는 버튼으로 유지
- 법인과 개인은 작은 텍스트 라벨로 구분
- `기타 주주`는 주요주주 목록과 분리해 하단 한 줄로 표시
- 상위 5명까지만 기본 노출

수평 막대 기준:

- 각 막대는 전체 발행주식 100%를 기준으로 그린다
- 최대값을 100%로 재정규화하지 않는다
- 수치가 작더라도 실제 지분율 관계가 왜곡되지 않아야 한다
- 막대만으로 값을 전달하지 않고 숫자를 함께 표시한다

### 4.4 본문 오른쪽: 기업 건전성 요약

오른쪽 영역은 세 개의 명확한 그룹으로 구성한다.

#### 주주환원

- 주당배당금
- 배당성향
- 단위를 명시하고 미제공 값은 `-`로 표시

#### 감사

- 감사의견을 가장 먼저 표시
- 회계감사인은 보조 정보로 표시
- 감사의견 원문을 임의로 축약하거나 등급화하지 않음

#### 핵심 재무비율

- 부채비율
- 총자산영업이익률
- ROE
- 값에 `%` 단위를 명시
- API가 이미 백분율 단위 값을 제공하는지 구현 전 확인
- 동일 단위를 중복 곱하거나 변환하지 않음

표현 원칙:

- 내부 박스 테두리를 최소화하고 얇은 행 구분선 사용
- 라벨과 값을 좌우 정렬한 정의 목록 형태 사용
- 숫자 크기와 소수점 자릿수를 일관되게 맞춤

### 4.5 출처 및 기준

- 카드 하단에 출처와 기준 보고서를 한 줄로 표시
- 예: `DART · 2025 사업보고서 기준`
- 정정 공시로 값이 바뀔 수 있다는 안내는 정보 아이콘 툴팁으로 이동
- 긴 안내 문장이 본문 시선을 방해하지 않도록 함

## 6. 시각 스타일

### 컨테이너

- 외곽 카드 하나만 유지
- 내부 중첩 카드의 배경, 그림자와 반복 테두리 제거
- 배경은 흰색 또는 기존 프로필의 중립색 사용
- 헤더, 요약 띠, 본문, 출처 사이만 얇은 구분선 적용

### 색상

- 기본 텍스트: 기존 `#101828` 계열
- 보조 텍스트: 기존 `#667085` 계열
- 주요주주 막대: 브랜드 파란색 한 계열의 명도 차이 사용
- 개인·법인 구분은 색상뿐 아니라 텍스트로 표시
- 감사의견과 재무비율에 의미가 검증되지 않은 빨강·초록을 사용하지 않음

### 타이포그래피

- 영역 제목: 기존 프로필 섹션 제목과 동일
- 핵심 신호 값: 18~22px
- 주요주주와 건전성 값: 14~16px
- 출처와 기준: 11~12px
- 숫자는 가능한 한 tabular number 스타일 사용

### 간격

- 외곽 패딩: 데스크톱 24px, 모바일 16px
- 핵심 신호 사이: 16~24px
- 주요주주 행 높이: 최소 44px
- 우측 정의 목록 행 높이: 최소 44px

## 7. 반응형 계획

### 821px 이상

- 핵심 신호 4열
- 본문 좌우 55:45
- 주요주주와 건전성 정보를 동시에 표시

### 561~820px

- 핵심 신호 2×2
- 본문을 세로 배치
- 주요주주 구조를 먼저 표시
- 기업 건전성 그룹을 아래에 표시

### 560px 이하

- 핵심 신호를 가로 스크롤이 없는 2열로 표시
- 주요주주 행은 `순위·이름 / 지분율` 구조로 압축
- 수평 막대는 이름 아래 전체 너비 사용
- 건전성 항목은 한 줄 정의 목록으로 표시
- 모든 텍스트와 버튼이 카드 밖으로 넘치지 않아야 함

### 360px 이하

- 핵심 신호를 1열 또는 2열 중 실제 콘텐츠 폭에 맞게 전환
- 긴 주주명은 최대 2줄 허용
- 지분율 숫자는 줄바꿈하지 않음
- 가로 스크롤을 허용하지 않음

## 8. 데이터 처리 기준

### 주요주주 집중도

- `기타 주주`를 제외한 표시 대상 주요주주의 지분율 합계로 계산
- 합계에 포함된 주주 범위를 툴팁으로 설명
- 중복 주주를 정규화한 뒤 계산
- 합계가 100%를 초과하면 계산값을 숨기고 데이터 검증 상태 표시

### 단위

- 모든 지분율과 재무비율에 `%` 단위 표시
- 주당배당금은 통화 단위를 확인해 표시
- API 원본이 이미 백분율인지 소수 비율인지 테스트 데이터로 확인

### 누락값

- 값이 없거나 해석할 수 없으면 `-`
- `0`이 실제 값인지 미제공 대체값인지 필드별로 확인
- 0이 유효한 값이면 `0` 또는 `0%`로 표시

### 과도한 해석 방지

- 임의의 점수, 등급 또는 종합 평가를 만들지 않음
- 업종 평균이나 전년 비교 데이터가 없으면 좋음·나쁨을 판정하지 않음
- 감사의견은 원문 값을 유지

## 9. 상호작용

- 주요주주명 클릭: 기존 주주 상세 모달 표시
- 우측 상단 더보기: 기존 통합 DART 상세 모달 표시
- 정보 아이콘 hover/focus: 계산 범위, 보고서 기준, 정정 가능성 설명
- 키보드로 모든 주주 버튼과 더보기 아이콘에 접근 가능
- 툴팁은 hover하지 않았을 때 숨김

## 10. 접근성

- 전체 영역은 `기업 심화 정보` 제목으로 식별
- 핵심 신호는 의미 있는 정의 목록(`<dl>`) 사용
- 주요주주 순위는 순서 있는 목록(`<ol>`) 사용
- 수평 막대는 장식으로 숨기고 지분율을 텍스트로 제공
- 정보 아이콘에 명확한 `aria-label` 제공
- 모달을 연 버튼으로 포커스가 정상 복귀해야 함
- 텍스트 및 핵심 수치의 명암비는 WCAG AA 수준 유지
- 200% 확대에서도 내용 겹침과 가로 스크롤이 없어야 함

## 11. 구현 계획

### 0단계: 데이터 계약 확인

- ownership, dividend, audit, ratios 실제 응답 샘플 확인
- 각 비율의 단위와 소수점 의미 확인
- 기타 주주와 주요주주 중복 처리 확인
- 0과 미제공 값 구분 확인

### 1단계: 파생 데이터 함수 작성

- 주요주주 정렬 및 상위 5명 추출
- 기타 주주 분리
- 주요주주 집중도 계산
- 비율 포맷과 단위 포맷 함수 정의
- 잘못된 합계를 방어하는 검증 로직 추가

### 2단계: 마크업 재구성

- 새로운 헤더와 기준 정보 구성
- 핵심 신호 요약 띠 추가
- 주요주주 순위 목록 구현
- 주주환원·감사·재무비율 정의 목록 구현
- 기존 상세 모달 버튼과 데이터 속성 유지

### 3단계: 스타일 재작성

- 중첩 카드 스타일 제거
- 데스크톱 55:45 본문 구성
- 주요주주 수평 막대와 숫자 정렬
- 기업 건전성 정의 목록 스타일 적용
- 기존 프로필 디자인 토큰 재사용

### 4단계: 반응형 적용

- 820px 이하 세로 전환
- 560px 이하 요약 신호 2열과 압축 행 적용
- 360px 긴 이름과 단위 줄바꿈 검증

### 5단계: 접근성 및 상호작용 검증

- 주주 상세 모달 연결 확인
- DART 통합 상세 모달 연결 확인
- 툴팁 hover/focus 상태 확인
- 키보드 이동과 모달 포커스 복귀 확인

### 6단계: 테스트 및 배포

- 렌더링 문자열 및 파생 계산 단위 테스트
- 누락·0·비정상 합계 테스트
- 1159px, 919px, 768px, 560px, 360px 시각 QA
- JavaScript 문법 검사와 전체 관련 테스트 실행
- 운영 서버 백업 후 배포
- 공개 페이지에서 최종 계산값과 반응형 확인

## 12. 완료 기준

- 첫 화면에서 최대주주, 집중도, 감사의견, 부채비율을 바로 확인할 수 있다.
- 주요주주가 지분율 순으로 정렬되고 수치와 막대가 일치한다.
- `기타 주주`가 주요주주와 명확히 구분된다.
- 배당, 감사, 재무비율이 서로 다른 의미 그룹으로 읽힌다.
- 모든 비율에 단위가 표시되고 원본 단위가 왜곡되지 않는다.
- 내부 중첩 카드 그림자와 반복 테두리가 제거된다.
- 768px, 560px, 360px에서 가로 스크롤이 없다.
- 누락값, 실제 0, 잘못된 합계가 각각 정의된 방식으로 처리된다.
- 주주 상세 및 DART 통합 상세 모달 기능에 회귀가 없다.
- 키보드 접근과 툴팁 focus 동작이 정상이다.
